What the SDK documents

The public API is organized around the work a reviewer or researcher needs to audit: input preparation, fitting, extraction, uncertainty review, figure generation, and interoperable exports.

Data

mSigPortal and local spectra

Project, sample, signature, exposure, MAF, SBS96, and matrix import helpers.

Fit

Known-signature workflows

NNLS refitting, cohort workflows, panel/WES evidence tiers, and reporting outputs.

Extract

Browser-sized NMF

Rank selection, extracted signatures, exposure heatmaps, worker execution, and handoffs.

QC

Uncertainty and evidence

Mutation burden, missing contexts, residuals, bootstrap intervals, ambiguity, and warnings.

Figures

Report-ready visualization

SBS96 profiles, exposures, residuals, similarity heatmaps, NMF profiles, and report panels.

Interop

External package handoffs

SigProfiler, deconstructSigs, sigminer, MuSiCal, Pyodide, WebR, JSON, TSV, and archives.
